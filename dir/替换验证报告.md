## 推送逻辑标准验证 (基于 tmp.txt)

### 步骤 1: 浏览器建立连接
**标准**: 页面加载时，通过 new WebSocket("ws://...") 与后端建立持久连接
**实现**: ✅ NettyWebSocketServer 监听 8081 端口，支持 ws://host:8081/ws 连接

### 步骤 2: 网关/负载均衡
**标准**: 转发 WebSocket 握手请求到具体的 IM 服务实例
**实现**: ✅ WebSocketHandshakeHandler 提取握手参数，支持 token 和 deviceId 传递

### 步骤 3: IM 服务接收握手并认证
**标准**: 接收握手，完成用户认证（通过 URL 参数 token），将 userId 与 WebSocketSession 绑定并存入内存 Map 或 Redis
**实现**: 
- ✅ WebSocketHandler.handleLogin() 方法处理登录
- ✅ ImSessionManager.bind() 方法绑定 userId 与 Channel
- ✅ 使用 Redis 存储在线状态 (im:online:user:{userId})
- ✅ 使用内存 Map (userChannels) 存储用户会话

### 步骤 4: 用户 A 发送消息
**标准**: 通过 HTTP 或 WebSocket 将消息发送到 IM 服务
**实现**: 
- ✅ WebSocketHandler.handlePrivateMessage() 处理 WebSocket 消息
- ✅ ImController.sendMessage() 处理 HTTP 消息

### 步骤 5: IM 服务处理消息
**标准**: 
- ① 校验消息
- ② 存储消息到数据库（持久化）
- ③ 触发推送逻辑

**实现**: 
- ✅ ImMessageServiceImpl.sendPrivateMessage() 校验消息
- ✅ 保存到 ImPrivateMessage 表
- ✅ 调用 messagePushService.pushToUser() 触发推送

### 步骤 6: 推送逻辑
**标准**: 根据接收方 userId，从在线会话 Map 中取出 WebSocketSession，通过该会话发送消息
**实现**: 
- ✅ ImSessionManager.sendToUser() 从 userChannels 获取会话
- ✅ 通过 Channel 发送 TextWebSocketFrame
- ✅ 支持多设备推送和排除指定设备

### 步骤 7: 用户 B 浏览器接收消息
**标准**: onmessage 事件触发，前端解析消息并更新页面 DOM（无需刷新）
**实现**: 
- ✅ 使用 ImWsEnvelope 统一消息格式
- ✅ 消息类型: MESSAGE_MESSAGE_PUSH
- ✅ 数据格式: JSON，包含 type, code, message, timestamp, data
